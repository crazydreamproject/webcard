Concept

Stack, Cards and Backgrounds

WebCard has basic concept of "Card", where you can draw picture, locate buttons and text fields. Think if it as a canvas to play with. You can create multiple cards and go forth or backword by pressing right and left arrow key. All of those cards you created is bundled as a "Stack". You can save and load your stack from "File" pull-down menu. You can also group cards with "Background". Picture, buttons and text fields in background is shared and displayed on cards having same background.

Buttons and Text Fields

Buttons and text fields (or just field) are object where players can interact with. You can create new button or field from "Objects" pull-down menu. Buttons and fields have style you can change by double-click specific button or field. You can write script for specific button and field to do what you want, like going to next card when "Next" button is pressed.

Browse and Edit Mode

In "Tools" pallete, you can find "Browse" and "Edit" button on the top. Press "Edit" or "Browse" button to be in that mode. On Edit mode, you can move buttons and fields around in card and edit by double-clicking it. On "Browse" mode, buttons and fields are sticked and will not move, and clicking button will fire the script you wrote for that button.

Auto Save

Opened stack is auto saved to your browser's local storage. When you go to Develop, last opened stack will be automatically loaded. If you want to start new stack, select "New Stack..." in "File" menu.

Scripting

How it works

Script is attached to object, like button, field, card, background and stack. Double click button or field object, or select Info... from "Object" pull-down menu to find Script button to start writing script. Script begins with "on" keyword, followed by event message, and ends with "end" keyword followed by same event message. Below is an example of script to run when mouse is clicked up on object. 


    on mouseUp
        go next card
    end mouseUp
Event message is a message sent to object. Here is list of event messages.

message name description
mouseDown When mouse button is pressed
mouseStillDown When mouse button is long pressed
mouseUp When mouse button is pressed and released
mouseDoubleClick When mouse button is double clicked
mouseEnter When mouse cursor is moved into object
mouseLeave When mouse cursor is moved out of object
mouseWithin When mouse cursor is within the object
arrowKey When arrow key is pressed. It takes one argument containing pressed key direction.
keyDown When any keyboard key is pressed
commandKeyDown When control key is pressed
returnKey When Return key is pressed
enterKey When Enter key is pressed
tabKey When Tab key is pressed
openCard When moving into card
closeCard When moving out to other card

Event message is first passed to current card's button or field if applicable, and then it is passed to current card, current background and finally to current stack, if none of them grabbed and run that specific message handler.

Script block, sandwitched by "on" <message> and "end" <message> is called message handler. You can write multiple message handlers in one script, like below. 


    on mouseEnter
       set name of me to "Mouse Inside"
    end mouseEnter
    on mouseLeave
        set name of me to "Mouse Outside"
    end mouseLeave
If there are same messages, only the first message handler will run.

Commands in script

Within the message handler, you can write commands to execute. See Scripting Reference for list of commands. Commands are executed from top line to bottom line sequentially, one at a time. You can modify this sequential flow by "if" and "repeat" keyword.

The "if" followed by conditional expression, and "then" will first evaluate that condition and if true, will go through list of commands in between "if" and "else" or "end if". If condition was false, then command lists between "else" and "end if" will be executed, if any. Example: 


    if field "name" is empty then
        ask "What is your name ?"
    end if

    if field "result" is "win" then
        show button "win"
    else
        show button "lose"
    end if

You can repeat block of commands, using "repeat" keyword. For example: 


    repeat 3 times
        beep
    end repeat

    repeat with x = 1 to 5
        put x*x into line x of field 1
    end repeat
(more to come about repeat)

When message handler reaches end of commands, message handler is done, and event message is consumed. However if you use "pass" keyword, it will quit the command execution in current message handler, and send event massage to next object applicable.

Scripting Reference

Abbreviation

Some keyword can be abbreviated.

Keyword Abberbiated

Comments

line that starts with # or // will be interpreted as comments for human readability and will cause no effect.
Example: 


    on mouseUp
        # This line is a comment. Check mouseUp message handler is called
        put "mouseUp is called" into fld 1
    end mouseUp


Number Literals

Number literals are just any numbers like 42 and 3.14 in the example below. 


    put 42 into field "answer"
    put 3.14 into field "PI"


String Literals

String literals are sentences like "quick brown fox" and "pangram" inside double quotes "" in the example below. 


    put "quick brown fox" into field "pangram"


Variables

Variables are used to hold values and to use it later in the script. To create variable, use put command.
In the example below, result is variable that holds a value, 42. 


    put 42 into result
There are two types of variables, local variables and global variables. Above is an example of "local" variables where only within that message handler can access to its value. To make it global, use global keyword to define. In below example, score and nextcard is global variables. 

    on mouseUp
        global score, nextcard
        put 42 into score
        if score > 40 then
            put "newRecord" into nextcard
        else
            put "showRanking" into nextcard
        end if
    end mouseUp
Later in another message handler of same or different object, these score and nextcard can be referenced. 

    on arrowKey
        global score, nextcard
        put score into field "currentRecord"
        go to card nextcard
    end arrowKey


Properties

Objects like button, field, card, background and stack have properties. You can get or set (if applicable) properties using get and set command. For example, to access name property of a button: 


    on mouseUp
        get name of btn 1
        # get will store the result in special variable called 'it'.
        put it into fld "btnName"
        # same thing in one action
        put name of btn 1 into fld "btnName"
        set name of btn 1 to "Renamed Btn Name"
    end mouseUp

Containers

Containers are places where you can put things in. Below are container types:

Some examples: 


    on mouseUp
        put "hello" into fld "salute"

go, push, pop, help, arrow key
syntaxdescriptionexample

These navigation commands let's you move from current card to another card.


Action Commands

put, delete, enable, disable, choose, click at, drag from, type
syntaxdescriptionexample
put <expr> Put some value to message box put "hello"
put <expr> <preposition_pointer> <variable> Put some value to variable put 42 into answerToEverything
put <expr> <preposition_pointer> <container> Put some value to container put "The" before line 1 of fld 1
delete <part_object_clause> Delete part object specified delete fld 3 of card 2
enable <button_object_clause> Enable button object specified enable btn 2 of card 2
disable <button_object_clause> Disable button object specified disable btn "OK"
choose <tool_type> Choose specified tool in Tool Palette choose browse tool
click at <expr>, <expr> Perform click at given location click at 100, 100, click at loc of btn 1
drag from <expr>, <expr> to <expr>, <expr> Perform drag from given location to location drag from 100, 100 to loc of btn 1
type <expr> Perform keyboard type of text type "hello"

You can combine these for example: 


    on mouseUp
        choose line tool
        drag from 100, 100 to 200, 200
        choose browse tool
    end mouseUp
to perform drawing from script! enable and disable will show objects, but in gray out if disabled. Also enabled property is set to true if enabled, and false if disabled.

Sound Commands

beep, play, play stop, speak, stop speach
syntaxdescriptionexample
beep [<expr>] Beep sound <expr> times (default 3 times) beep 1
play <string literal> Play sound with "instrument" play "boing"
play <string literal>, <string literal> Play sound "keys" with "instrument" play "flute", "c4 d e f"
play <string literal>, tempo <expr>, <string literal> Play sound "keys" with tempo <expr> with "instrument" play "harpsicode", tempo 100, "c4 d e f g a b c5"
play stop Stop playing sound started by play command play stop
stop sound Stop all sounds stop sound
speak <string literal>, with <voice type> Speak sentence with given voice (see below) speak "Bye-Bye" with female voice
stop speach Stop speak command stop speach

<voice type> could be male voice, female voice, neuter voice, or any installed voice with double quote, for example: 


    on mouseUp
        speak "Good Morning" with voice "Whisper"
    end mouseUp
Sound commands are now under development and currently not functional.

Arithmatic Commands

add, subtract, multiply, divide
syntaxdescriptionexample
add <expr> to <container>|<variable> Add <expr> value to <container> or <variable> add 5 to fld 2
subtract <expr> from <container>|<variable> Subtract <expr> value from <container> or <variable> subtract 3 from var2
multiply <container>|<variable> by <expr> Multiply <container> or <variable> by <expr> value multiply var2 by id of btn "mul"
divide <container>|<variable> by <expr> Divide <container> or <variable> by <expr> value divide message box by fld 2

Directly apply arithmatic operations to <container> or <variable>. put can be used to do same thing (with longer words). 


    on mouseUp
        add 5 to fld 2
        # below is equivalent of above
        put (fld 2) + 5 into fld 2
    end mouseUp

Display Commands

visual, flash, answer, ask, lock screen, unlock screen
syntaxdescriptionexample
visual|visual effect <visual effect clause> Apply visual effect, usually to current card visual effect wipe left slowly
flash (<expr>) flash, usually current card <expr> times (default 3 times) flash 5
answer <expr> [ with <string literal> [ or <string literal> [ or <string literal> ] ] ] Prompts user with multiple (up to 3) choices. Result in it variable. answer "Beef or Chicken?" with "Beef" or "Chicken" or "Vegi"
ask <expr> [ with <expr> ] Ask user with default string. Result in it variable. ask "Where do you live?" with "On Earth"
lock screen Stop changing visual of current card lock screen
unlock screen|unlock screen with visual effect <visual effect clause> Redraw current card with specified visual effect unlock screen with visual effect checkerboard

Usually, lock screen and unlock screen is used as pair, with some visual change on card added in between. 


    on mouseUp
        lock screen
        go next card
        unlock screen with visual effect barn door open
    end mouseUp
Caveat, currently, ask and answer will return result to it variable AFTER user input to prompt is made, but script will run async with prompt. So below will NOT put result in it variable at this timing. 

    on mouseUp
        answer "Black or White?" with "Black" or "White"
        # below will not reflect above result since at this time, user has not yet input to prompt.
        put it into fld 1
    end mouseUp

Object Manipulation Commands

get, set, show, hide
syntaxdescriptionexample
get <property clause> Obtain <property> and put into it variable get visible of fld 1
set <property clause> to <expr> Set <property> to <expr> set textFont of fld 1 to "Helvetica"
hide <object clause>|menuBar|titleBar Hide <object> so it can not be seen. see visible property. hide card button "show result"
show <object clause> Show <object> so it can be seen. see visible property. show bg field "status"

hide will set visible property of object to false and show will set visible property of object to true. 


    on mouseUp
        hide btn 1
        # below will put false
        put visible of btn 1 into fld 1
    end mouseUp
